# LLM Clustering для контакт-центра

Проект для кластеризации обращений клиентов в контакт-центр с использованием исключительно LLM.

## Структура проекта

```
llm_clustering/
├── src/
│   └── llm_clustering/
│       ├── __init__.py
│       ├── config/          # Конфигурация и настройки
│       ├── llm/             # LLM провайдеры (OpenAI, Anthropic)
│       ├── clustering/      # Основная логика кластеризации
│       ├── core/            # Ядро приложения
│       └── utils/           # Утилиты (логирование и т.д.)
├── tests/                   # Тесты
├── ai_doc/                  # Документация
├── ai_experiments/          # Эксперименты и тесты функциональности
└── ai_data/                 # Данные, логи, промежуточные результаты
```

## Установка

```bash
# Создать виртуальное окружение
python3.12 -m venv venv
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
# или
pip install -e ".[dev]"
```

## Конфигурация

Скопируйте `env.example` в `.env` и заполните API ключи:

```bash
cp env.example .env
```

**Важно**: Проект настроен на использование **Qwen 3 Next** через **OpenRouter** по умолчанию.
Убедитесь, что в `.env` указан `OPENROUTER_API_KEY`.

## Основные компоненты

- **LLM Providers**: Абстракция для работы с различными LLM провайдерами
- **Clusterer**: Основной класс для кластеризации обращений
- **Config**: Централизованная конфигурация через Pydantic Settings
